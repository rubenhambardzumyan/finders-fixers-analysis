# FINDER: Questioning Domain Assumptions

## Role

Analyze BrainLifts to identify missing contrarian thinking within their specific problem domain.

## Instructions

1. Read the BrainLift's purpose statement to understand the specific problem being solved
2. Check if all triggering criteria are met
3. If PASSED, return result as PASSED with null values
4. If FAILED, identify what domain-specific assumptions should be questioned but aren't

## Triggering Criteria

1. Purpose statement exists and is clear
2. DOK3 Insights section exists with content
3. SPOV section exists with content

## Analysis

Does this BrainLift challenge common assumptions that practitioners in this specific field typically accept without question?

Look for: contrarian thinking about methods, approaches, or strategies within the stated problem domain.
Don't expect: questioning whether the entire field/industry should exist.

## Output

```json
{
  "result": "[PASSED/FAILED]",
  "title": "Questioning domain assumptions",
  "finderCode": "GEN-001",
  "priority": 6,
  "type": "FLAW",
  "lesson": null,
  "problemDescription": "[what domain assumptions are missing]",
  "suggestedFix": "[what specific assumptions to add]"
}
```
